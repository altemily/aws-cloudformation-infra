# 🌩️ AWS CloudFormation – Automação de Infraestrutura

Este repositório documenta minha prática no desafio **Automatizando com AWS CloudFormation** da DIO.  
O objetivo é aplicar conceitos de **Infraestrutura como Código (IaC)** usando CloudFormation, criando stacks reutilizáveis e documentando o aprendizado de forma clara.

---

## 🚀 Passo a passo da prática

1. **Criação da Stack no console AWS**  
   - Criei uma stack a partir de um template em YAML.  
   - Testei recursos como **S3**, **Lambda** e **EC2**.  
   - Validei os templates antes do deploy para evitar falhas.  

2. **Gerenciamento de recursos**  
   - Experimentei atualizar e deletar stacks de forma controlada.  
   - Entendi como mudanças no template refletem diretamente na infraestrutura.  

3. **Documentação do aprendizado**  
   - Anotações e insights foram registrados em [insights.md](./insights.md).  
   - Templates organizados na pasta [examples](./examples).  

---

## 💡 Principais aprendizados

- **Automação** reduz erros humanos e aumenta a confiabilidade.  
- **IaC** facilita a replicação de ambientes (dev, staging, prod).  
- **Validação** dos templates é essencial para evitar falhas em produção.  
- **CloudFormation** integra versionamento e governança, essencial em equipes grandes.  

Mais detalhes estão em: [insights.md](./insights.md).
